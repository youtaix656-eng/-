# るるくる セラピスト業務マニュアル（個人用アプリ）

るるくる（もみほぐし専門店）のセラピストが、業務手順・接客ルール・施術内容を
**スマホで片手でサッと確認する**ための個人用ウェブアプリです。

- スマホファースト（1カラム・大きめの文字・ゆったりした行間）
- オフラインでも見られる（PWA・ホーム画面に追加できる）
- 「チェック済み」「お気に入り」はこの端末（ブラウザ）だけに保存されます
- サーバーやデータベースは不要。中身はすべてこのプロジェクト内のファイル

> ⚠️ このアプリは「入れ物」です。実際のるるくるの業務内容・社内ルールは、
> 店舗や研修で配布される資料に必ず従ってください。中身（マニュアル本文）は
> 研修で学んだ内容をご自身で入力する前提です。

---

## 1. パソコンで動かす（開発モード）

初回だけ、必要なものをインストールします。

```bash
npm install
```

起動：

```bash
npm run dev
```

ブラウザで **http://localhost:3000** を開くと表示されます。
（スマホと同じ端末のブラウザで確認するのがおすすめです）

本番用に書き出して確認する場合：

```bash
npm run build
npm run start
```

PWA（オフライン閲覧・ホーム画面追加）は `npm run build` → `npm run start` の
本番モードで有効になります。

## 2. 画面の構成

| 画面 | URL | 内容 |
| --- | --- | --- |
| ホーム | `/` | 検索バー・カテゴリ一覧・よく見る項目（お気に入り） |
| カテゴリ | `/category/[slug]` | そのカテゴリの項目をアコーディオン表示 |
| 詳細 | `/item/[id]` | 見出し・本文・手順・注意点・関連項目 |
| チェックリスト | `/checklist` | 出勤／施術前後／退勤のチェック（日付が変わると自動リセット） |
| 問題 | `/quiz` | 自主基準の一問一答・正誤問題（○×）で理解度チェック |
| 自主基準テスト | `/test` | 本番形式の全29問。回答するとすぐ解答・解説を表示 |
| 用語集 | `/glossary` | 五十音順・頭文字ジャンプ付き |

## 3. データ（マニュアルの中身）の追加・編集方法

**編集する場所はすべて `src/data/` フォルダの中です。**
プログラミングの知識がなくても、既存のサンプルをコピーして書き換えれば追加できます。

### ファイルの役割

| ファイル | 中身 |
| --- | --- |
| `src/data/categories.ts` | カテゴリ（大分類）の一覧 |
| `src/data/items.ts` | 各項目の詳細（手順・注意点など）※ ここを一番よく編集します |
| `src/data/checklists.ts` | チェックリストの項目 |
| `src/data/glossary.ts` | 用語集 |
| `src/data/quiz.ts` | 問題（一問一答 `qaList` と 正誤問題○× `oxList`） |
| `src/data/standardsTest.ts` | 自主基準テスト（全29問）。`correct` は正解の選択肢番号（0始まり） |
| `src/data/types.ts` | データの「型」の説明（触らなくてOK・迷ったら読む） |

### 例：新しい項目を1つ追加する

`src/data/items.ts` を開き、既存のサンプル（`{ ... }` のかたまり）を
1つコピーして、いちばん最後の `]` の直前に貼り付けます。そして中身を書き換えます。

```ts
{
  id: "attendance-new-rule",        // 他と重複しない英数字の名前（自由につけてOK）
  categorySlug: "attendance",       // どのカテゴリに入れるか（categories.ts の slug）
  title: "新しい項目のタイトル",       // 見出し
  summary: "一覧に出る短い説明",       // なくてもOK（省略可）
  body: "本文をここに書きます。\n改行も使えます。", // なくてもOK
  steps: [                          // 手順（なくてもOK）
    { text: "ステップ1の内容" },
    { text: "ステップ2の内容", note: "補足メモ" },
  ],
  warnings: [                       // 注意点（なくてもOK）
    { level: "warning", text: "黄色で表示される注意" },
    { level: "danger",  text: "赤色で表示される禁忌・厳守事項" },
  ],
  related: ["attendance-clock-in"], // 関連項目の id（なくてもOK）
  tags: ["キーワード1", "キーワード2"], // 検索に引っかかる言葉（なくてもOK）
},
```

**ポイント**

- 各行のうしろの `,`（カンマ）を消さないでください。
- 文字は必ず `"` … `"`（ダブルクオート）で囲みます。
- `warnings` の `level` は
  - `"warning"` = アンバー（注意）
  - `"danger"` = レッド（禁忌・厳守）
- `summary` `body` `steps` `warnings` `related` `tags` は **不要なら行ごと消してOK**。
- 保存すると、開発モード（`npm run dev`）なら自動で画面に反映されます。

### 例：カテゴリを追加する

`src/data/categories.ts` にサンプルをコピーして追記します。
`icon` は [lucide-react](https://lucide.dev/icons/) のアイコン名（英語）を書きます。
（例：`"Clock"`, `"Hand"`, `"Sparkles"`）

### 例：チェックリスト・用語を追加する

`src/data/checklists.ts` / `src/data/glossary.ts` の中の
サンプルをコピーして書き換えるだけです。用語集は `reading`（よみがな・ひらがな）で
五十音順に自動で並びます。

### 例：問題（一問一答・○×）を追加する

`src/data/quiz.ts` を編集します。

- 一問一答は `qaList` に追記：`{ id, q: "問い", a: "答え", ref: "第X条" }`
- 正誤問題は `oxList` に追記：
  `{ id, statement: "正誤を問う文", answer: true, explanation: "解説", ref: "第X条" }`
  - `answer` は正しければ `true`（○）、誤りなら `false`（×）

## 4. スマホのホーム画面に追加する（PWA）

1. 公開したアプリ（または `npm run start` で起動したもの）をスマホのブラウザで開く
2. ブラウザのメニューから「ホーム画面に追加」を選ぶ
3. アイコンから起動すると、アプリのように全画面で使えます（オフライン可）

---

## 5. こまったときは

- 画面が真っ白／エラーが出る → だいたい `src/data/` のどこかで
  `,` や `"` が抜けています。直前に編集した場所を見直してください。
- 変更が反映されない → `npm run dev` を再起動してみてください。
- お気に入り・チェックが消えた → ブラウザのデータを消すとリセットされます
  （この端末のブラウザだけに保存されているためです）。

## 技術メモ（開発者向け）

- Next.js 14（App Router）+ TypeScript + Tailwind CSS
- アイコン：lucide-react
- 状態保存：localStorage（`src/lib/storage.ts`）
- 検索：全コンテンツ横断の簡易あいまい検索（`src/lib/search.ts`）
- PWA：`public/manifest.json` + `public/sw.js`（`ServiceWorkerRegister` で登録）
- アイコン生成：`node scripts/generate-icons.js`（外部依存なし）
