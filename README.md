# 鍼灸国試 対策アプリ

鍼灸国家試験対策の学習アプリです。過去問題集の内容をベースにした問題演習を、
**サーバー不要・ブラウザ内で完結**する形で行えます。React + Vite + Web Speech API で構築しています。

## 主な機能

| 機能 | 内容 |
| --- | --- |
| 🖊 一問一答 | 科目別に分類。○×形式・四択形式に対応。解答後すぐに正誤と解説を表示。 |
| 🔁 間違えた問題の自動復習 | 誤答した問題を自動でストック。**間隔反復（SM-2 アルゴリズム）** で再出題。復習時に「むずかしい / ふつう / かんたん」を自己評価して間隔を最適化。「間違えた問題だけ解く」モード。 |
| 🎧 音声学習モード | 間違えた問題を音声合成で読み上げ。連続再生・一時停止・スキップ・再生速度調整に対応。「問題文 →（数秒の間）→ 正解と解説」の順で自動再生。再生中は **Screen Wake Lock** で画面スリープを抑制。 |
| 📝 模擬試験 | 問題数・制限時間を選んで通し演習。終了後に正答率・合格ライン判定（60%）と科目別内訳を表示。 |
| 📊 弱点分析ダッシュボード | 科目別正答率をバーグラフで表示。苦手科目を強調。直近14日の学習量も表示。 |
| 📌 メモ・付箋 | 間違えた問題に覚え方やメモを残せる。メモ一覧からまとめて確認・編集。 |
| 🖼 画像問題 | 経穴図・レントゲンなどの図表問題に対応（画像URL / データURIを持てるスキーマ）。 |
| 📷 OCR取り込み | 本のページ写真から文字を抽出する下処理ツール（`tesseract.js` を使用時にCDNから読み込み）。 |
| 💾 バックアップ / 復元 | 問題・進捗・メモ・設定を1ファイルに書き出し、別端末で復元可能。解答が一定数貯まると保存を促し、起動時の自動バックアップにも対応。 |
| 📲 PWA | ホーム画面に追加してアプリのように起動。Service Worker により一度開けばオフラインでも利用可能。 |

## データ保存

- すべてブラウザ内（**IndexedDB**）に保存され、**サーバーには送信されません**。
- 進捗・誤答履歴・メモ・設定は端末を閉じても保持されます。
- 旧バージョンで `localStorage` に保存したデータは初回起動時に自動移行します。
- 端末間で持ち運ぶ場合は、設定画面の「バックアップと復元」でファイルをやり取りします（クラウド保存やUSB経由。特定サービスへのAPI連携なしで完結）。

## 問題データの投入

- 初回はサンプル問題（6科目・約20問）で動作確認できます。
- 本番データは **設定画面から CSV / JSON でインポート** できます。

### CSV フォーマット

項目：`科目, 問題文, 選択肢, 正解, 解説`

```csv
科目,問題文,選択肢,正解,解説
経絡経穴概論,合谷穴が属する経絡はどれか。,手の陽明大腸経|手の太陰肺経|手の少陰心経|手の厥陰心包経,1,合谷は手の陽明大腸経の原穴。
東洋医学概論,督脈は身体前正中線を上行する。,,×,督脈は後正中線を上行する。前正中線は任脈。
```

- **選択肢**：半角パイプ `|` 区切り。空欄にすると ○×（正誤）問題になります。
- **正解**：四択は番号（1〜4）または選択肢の文字列、○×は `○` / `×`（`正しい`/`誤り` なども可）。
- JSON は上記項目のオブジェクトの配列。`choices`（配列）・`answer`（0始まりインデックス）形式にも対応。

> 将来的に、本のページ写真を OCR で読み取ったテキストをこの形式に整形して取り込む運用を想定しています。

現在の問題データは設定画面から CSV / JSON でエクスポート（バックアップ）できます。

## 開発・ビルド

```bash
npm install      # 依存関係のインストール
npm run dev      # 開発サーバー（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果のプレビュー
npm test         # ユニットテスト（importer / srs）
```

`dist/` は静的ファイルのみで動作するため、GitHub Pages など任意の静的ホスティングに配置できます
（`vite.config.js` で相対パス `base: './'` を設定済み）。

## GitHub Pages への自動デプロイ

`.github/workflows/deploy.yml` を用意しています。`main` ブランチへ push すると、
テスト → ビルド → デプロイが自動実行されます。初回のみ次の設定が必要です。

1. GitHub リポジトリの **Settings → Pages** を開く
2. **Source** を **「GitHub Actions」** に設定する

以後、`main` への push ごとに `https://<ユーザー名>.github.io/<リポジトリ名>/` へ公開されます。

## テスト

純粋ロジック（CSV/JSONインポータ、SM-2の間隔計算）に Node 標準のテストランナーで
ユニットテストを用意しています（`npm test`）。CI（GitHub Actions）でもデプロイ前に実行されます。

## 技術構成

- **React 18** + **Vite**（外部サーバー・API課金なし）
- **Web Speech API（speechSynthesis）** による音声読み上げ
- 永続化は `localStorage`
- グラフは依存ライブラリなしの自前 SVG/CSS 実装

## ディレクトリ構成

```
src/
  components/   各画面（Home, Quiz, Review, AudioMode, Exam, Dashboard, Memos, Settings）
  lib/          ロジック（storage, srs, importer, speech, stats, useStore）
  data/         サンプル問題
  styles/       スタイル
```

## 対応・注意事項

- スマホ利用がメイン。レスポンシブ対応済み。
- 音声学習はアプリを開いた状態での利用を想定しています。端末の画面OFFやブラウザのバックグラウンド化により、OSの仕様で音声が停止する場合があります。
- 音声合成は端末・ブラウザに搭載された音声を使用します。日本語音声が入っていない環境では読み上げ品質が下がることがあります。
